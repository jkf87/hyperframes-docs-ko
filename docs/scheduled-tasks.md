# 예약 작업

`/loop` 명령어와 cron 스케줄링 도구를 사용하여 프롬프트를 반복 실행하거나 일회성 알림을 설정할 수 있습니다.

<Note>
  예약 작업은 Claude Code v2.1.72 이상이 필요합니다. `claude --version`으로 버전을 확인하세요.
</Note>

## 스케줄링 옵션 비교

| | [Cloud](/web-scheduled-tasks) | [Desktop](/desktop-scheduled-tasks) | [`/loop`](/scheduled-tasks) |
|---|---|---|---|
| 실행 위치 | Anthropic 클라우드 | 로컬 머신 | 로컬 머신 |
| 머신 켜짐 필요 | 아니오 | 예 | 예 |
| 세션 열림 필요 | 아니오 | 아니오 | 예 |
| 재시작 후 유지 | 예 | 예 | 아니오 (세션 범위) |
| 로컬 파일 접근 | 아니오 (fresh clone) | 예 | 예 |
| 최소 간격 | 1시간 | 1분 | 1분 |

**팁:** 머신 없이 안정적으로 실행해야 한다면 **Cloud 작업**을, 로컬 파일 접근이 필요하면 **Desktop 작업**을, 세션 중 빠른 폴링이 필요하면 **`/loop`**를 사용하세요.

## /loop로 프롬프트 반복 실행

### 고정 간격

```text
/loop 5m 배포가 완료되었는지 확인해줘
```

간격 단위: `s`(초), `m`(분), `h`(시), `d`(일). 초는 가장 가까운 분으로 반올림됩니다.

### Claude가 간격 선택

```text
/loop CI가 통과했는지 확인하고 리뷰 코멘트에 응답해줘
```

간격을 생략하면 Claude가 매 반복 후 관찰 결과에 따라 1분~1시간 사이의 지연을 동적으로 선택합니다.

### 기본 유지보수 프롬프트

```text
/loop
```

프롬프트 없이 실행하면 기본 유지보수 프롬프트가 동작합니다: 미완료 작업 계속 → PR 리뷰 코멘트 처리 → CI 실패 확인 → 정리 작업 수행.

### loop.md로 커스텀

`loop.md` 파일로 기본 프롬프트를 대체할 수 있습니다:

| 경로 | 범위 |
|---|---|
| `.claude/loop.md` | 프로젝트 수준 (우선) |
| `~/.claude/loop.md` | 사용자 수준 |

## 일회성 알림

```text
오후 3시에 릴리스 브랜치를 푸시하라고 알려줘
```

```text
45분 후에 통합 테스트가 통과했는지 확인해줘
```

## 예약 작업 관리

자연어로 작업을 조회하거나 취소할 수 있습니다:

```text
어떤 예약 작업이 있어?
```

```text
배포 확인 작업 취소해줘
```

내부적으로 다음 도구를 사용합니다:

| 도구 | 용도 |
|---|---|
| `CronCreate` | 새 작업 스케줄 |
| `CronList` | 모든 예약 작업 목록 |
| `CronDelete` | 작업 ID로 취소 |

## 작업 동작 방식

- 스케줄러는 매초 만료된 작업을 확인하고 저우선순위로 큐에 추가합니다.
- Claude 응답 중에는 아닌 턴 사이에 실행됩니다.
- 모든 시간은 로컬 타임존으로 해석됩니다.
- 지터(jitter): 반복 작업은 주기의 최대 10%(최대 15분) 지연으로 실행됩니다.
- 7일 만료: 모든 작업은 7일 후 자동 종료됩니다.
